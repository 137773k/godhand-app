你需要修改文件 sketches/010-realistic-hand-login/index.html，让它从一个纯视觉稿变成一个真正可用的手机登录页面。

## 严格要求：必须100%保留现有视觉风格
- 暗色深空背景 + 液态玻璃反光层
- 金色系配色 (#f5d79d, #e6ad54, #f8f3ea)
- HAND OF GOD 标题样式
- 9:19 竖屏手机框架
- 现有的所有 CSS 渐变、模糊、光晕效果
- 玻璃球装饰、灰尘粒子、雾化层
- 开始塑造 按钮的金色渐变样式
- 把未来拖到掌心里 副标题

## 需要添加的功能（不破坏现有样式）：

1. 真输入框：把装饰性的placeholder div改成真正的 input 元素，type=tel，添加中国手机号格式校验（1开头的11位数字），带 placeholder=手机号，inputmode=numeric

2. 验证码输入：在手机号下方加一个验证码输入框 + 发送验证码 按钮，按钮带60秒倒计时

3. 按钮三态：
   - 默认 disabled（灰色/降低透明度）
   - 手机号+验证码都合法后变为 active（当前金色渐变）
   - 点击后显示 loading spinner + 文字变 验证中...

4. 协议勾选：在按钮上方加 checkbox 同意用户协议和隐私政策，不勾选不能点按钮

5. 注册入口：按钮下方加 还没有账号？立即注册 链接

6. 错误提示：输入框下方加实时校验提示文字（红色小字），手机号格式不对时显示错误信息

7. Toast 提示组件：一个固定在顶部的 toast，用于显示成功/失败消息

8. 键盘适配：添加 viewport meta 和 CSS 确保移动端键盘弹起时页面自动上移，输入框不被遮挡

9. 交互反馈：
   - 输入框 focus 时底部金色发光线变亮
   - 按钮 hover 时微放大 + 光晕增强
   - 按钮 active 时缩小 + 变暗

10. JS 逻辑：写完整的 JavaScript 处理表单校验、验证码倒计时、按钮状态切换、模拟提交（2秒后随机成功/失败，显示 toast）

所有新增元素必须继承现有配色和风格，不能引入新的设计语言。改完以后用浏览器打开预览截图确认效果。
